## duck-gen JSON Config <Badge>feat</Badge>

Switched duck-gen configuration from TypeScript to JSON with a new `outputSource` field.

- Migrated config format from `duck-gen.config.ts` to `duck-gen.json`
- Added JSON Schema (`duck-gen.schema.json`) for editor autocomplete and validation
- Added `$schema` field support for VS Code IntelliSense
- Restructured configuration handling and API route processing

### Before

```ts title="duck-gen.config.ts"
export default {
  framework: 'nestjs',
  tsconfigPath: './tsconfig.json',
}
```

### After

```json title="duck-gen.json"
{
  "$schema": "node_modules/@gentleduck/gen/duck-gen.schema.json",
  "framework": "nestjs",
  "extensions": {
    "shared": {
      "tsconfigPath": "./tsconfig.json"
    }
  }
}
```

## Multiple Output Paths <Badge>feat</Badge>

Added support for **multiple output directories** in code generation.

- `outputSource` now accepts a `string` or `string[]`
- Generated types can be written to multiple locations simultaneously
- Useful for monorepo setups where server and client share types

```json title="duck-gen.json"
{
  "extensions": {
    "shared": {
      "outputSource": ["./generated", "../client/generated"]
    }
  }
}
```
